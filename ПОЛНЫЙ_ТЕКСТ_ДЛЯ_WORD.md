# ПОЛНЫЙ ТЕКСТ ДЛЯ ЗАМЕНЫ В WORD ДОКУМЕНТЕ

## ВВЕДЕНИЕ

В современном цифровом мире проблема фишинговых атак становится все более актуальной. Фишинг представляет собой метод мошенничества, при котором злоумышленники пытаются получить конфиденциальную информацию пользователей (пароли, данные банковских карт, личные данные) путем рассылки поддельных сообщений или создания поддельных веб-сайтов, имитирующих легитимные сервисы.

Актуальность проблемы подтверждается статистикой: по данным компании Netskope, количество фишинговых переходов выросло на 190% в 2024 году по сравнению с 2023 годом. Более 42% пользователей сталкивались с фишинговыми атаками, при этом 17% пользователей подвергались атакам через социальные сети, а 13% - через мессенджеры.

Для решения данной проблемы была разработана комплексная система автоматического определения фишинга с использованием технологий машинного обучения. Система предоставляет несколько интерфейсов доступа: веб-приложение, Telegram бот с Web App, Chrome расширение и REST API для интеграции с другими системами.

## РАЗДЕЛ 1. ОБОСНОВАНИЕ НЕОБХОДИМОСТИ РАЗРАБОТКИ СИСТЕМЫ

### 1.1 Актуальность проблемы фишинга в современном мире

В 2024 году проблема фишинга достигла критических масштабов. Согласно отчетам аналитических компаний, наблюдается стремительный рост количества фишинговых атак. Компания Netskope в своем исследовании сообщает о более чем 1000 фишинговых переходов в месяц на одного сотрудника, что на 190% больше, чем в предыдущем году.

Киберпреступники активно используют различные каналы для распространения фишинговых сообщений. По данным Microsoft, 42% всех фишинговых атак происходят через электронную почту. Значительная доля атак также приходится на социальные сети (17%) и мессенджеры (13%), особенно популярные в России платформы.

В России ситуация также остается критической. По данным RED, более 30% фишинговых атак в 2024 году были направлены на финансовые учреждения и крупные корпорации. При этом 28% всех фишинговых атак были связаны с использованием поддельных URL-адресов и фишинговых веб-сайтов.

Важной проблемой является также рост числа фишинговых атак через мобильные устройства и мессенджеры, включая Telegram, где мошенники активно используют различные уловки для обмана пользователей.

### 1.2 Анализ существующих решений

В настоящее время на рынке представлено несколько категорий решений для защиты от фишинга:

**Антивирусные решения с модулями защиты от фишинга:**
- Norton 360 Deluxe: предлагает встроенный модуль Norton Safe Web, который проверяет URL на наличие в черных списках. Однако решение платное и требует подписки.
- Avast Online Security: расширение для браузера Avast, которое сканирует веб-страницы и проверяет URL-адреса, но не использует машинное обучение для анализа содержимого.

**Специализированные расширения для браузеров:**
- PhishNet: расширение для Chrome, использующее базы данных известных фишинговых сайтов для проверки URL.
- Защита от веб-угроз 360: расширение для Chrome, проверяющее URL через базу данных компании 360 Total Security.

Все эти решения имеют существенные ограничения: они полагаются исключительно на черные списки URL, что означает, что новые фишинговые атаки могут быть не обнаружены до тех пор, пока сайт не будет внесен в базу данных. Кроме того, существующие решения не анализируют содержимое текста, изображений или email-файлов с помощью машинного обучения.

### 1.3 Цель и задачи проекта

**Цель проекта:** Разработка комплексной системы автоматического определения фишинга с использованием технологий машинного обучения, предоставляющей пользователям удобные интерфейсы для проверки текстов, изображений и email-файлов через веб-приложение, Telegram бота и Chrome расширение.

**Задачи проекта:**

1. **Разработка модели машинного обучения:**
   - Сбор и объединение датасетов для обучения модели (текстовые сообщения, email-письма, URL-адреса)
   - Предобработка данных и извлечение признаков
   - Обучение моделей (Random Forest, Logistic Regression)
   - Оценка качества моделей и выбор оптимальной

2. **Разработка веб-приложения:**
   - Создание Flask API для обработки запросов на проверку
   - Разработка пользовательского интерфейса для проверки текста, изображений и .eml файлов
   - Интеграция OCR (Optical Character Recognition) для анализа изображений
   - Реализация парсера для .eml файлов

3. **Разработка Telegram бота:**
   - Создание Telegram бота с поддержкой команд
   - Интеграция Telegram Web App для работы внутри мессенджера
   - Реализация обработки текстовых сообщений, изображений и файлов

4. **Разработка Chrome расширения:**
   - Создание расширения для браузера Chrome (Manifest V3)
   - Реализация автоматического извлечения текста со страниц
   - Интеграция с API для проверки на фишинг

5. **Развертывание системы:**
   - Настройка Docker контейнера для развертывания
   - Развертывание на облачной платформе (Railway.app)
   - Настройка HTTPS для работы Telegram Web App

### 2.3 Разработка алгоритма работы системы

**Общая схема работы:**

1. **Пользователь отправляет данные для проверки:**
   - Через веб-интерфейс: текст, изображение или .eml файл
   - Через Telegram Web App: аналогично веб-интерфейсу
   - Через Chrome расширение: автоматическое извлечение текста со страницы

2. **Обработка данных на сервере:**
   - Для текста: извлечение признаков (наличие URL, email, телефонов), очистка и векторизация
   - Для изображений: извлечение текста с помощью OCR (Tesseract), затем обработка как текста
   - Для .eml файлов: парсинг email, извлечение тела письма, затем обработка как текста

3. **Анализ с помощью ML-модели:**
   - Векторизация текста с помощью TF-IDF
   - Добавление дополнительных признаков (URL, email, телефон)
   - Предсказание вероятности фишинга с помощью обученной модели Logistic Regression
   - Определение уровня риска:
     - Безопасно (0-30%)
     - Подозрительно (30-50%)
     - Опасно (50-70%)
     - Критично (70-100%)

4. **Формирование и отправка результата:**
   - Возврат JSON-ответа с результатами анализа
   - Отображение результатов в интерфейсе с визуальной индикацией

Система построена по модульной архитектуре с разделением на несколько основных компонентов:

   - Класс PhishingModel для загрузки и использования обученной модели
   - Методы для предобработки текста (clean_text)

   - Flask приложение с маршрутами для API и веб-интерфейса

3. **Утилиты (webapp/utils.py):**
   - Функции для извлечения URL, email, телефонов из текста

   - Обработчики команд (/start, /help)

Модуль машинного обучения инкапсулирован в отдельный класс, что позволяет легко переключаться между различными моделями или обновлять модель без изменения остального кода.

## РАЗДЕЛ 3. РЕАЛИЗАЦИЯ ПРОЕКТА

### 3.1 Разработка и обучение модели машинного обучения

Для создания модели машинного обучения использовались три датасета:
1. **Dataset_5971.csv** - датасет с текстовыми сообщениями, содержащий метки: smishing (фишинг), spam (спам), ham (безопасно)
2. **Phishing_Email.csv** - датасет с email-письмами, содержащий метки: phishing email (фишинг), safe email (безопасно)
3. **PhiUSIIL_Phishing_URL_Dataset.csv** - датасет с URL-адресами, содержащий бинарные метки (0 - безопасно, 1 - фишинг)

**Процесс обучения модели:**

1. **Загрузка и объединение датасетов:**
   - Загрузка каждого датасета с обработкой различных форматов данных
   - Приведение меток к единому формату (0 - безопасно, 1 - фишинг)
   - Объединение всех датасетов в единый датасет для обучения

2. **Предобработка данных:**
   - Очистка текста от лишних символов (сохранение @ и / для URL и email)
   - Нормализация пробелов
   - Приведение к нижнему регистру
   - Извлечение дополнительных признаков: наличие URL, email, телефона в тексте

3. **Векторизация текста:**
   - Использование TF-IDF Vectorizer с параметрами:
     - max_features=10000 - максимальное количество признаков
     - ngram_range=(1,2) - использование униграмм и биграмм
     - min_df=2 - минимальная частота документа
     - max_df=0.95 - максимальная частота документа
     - stop_words='english' - удаление стоп-слов

4. **Обучение моделей:**
   - **Random Forest Classifier:**
     - n_estimators=200 - количество деревьев
     - max_depth=25 - максимальная глубина деревьев
     - class_weight='balanced' - балансировка классов
   - **Logistic Regression:**
     - max_iter=2000 - максимальное количество итераций
     - class_weight='balanced' - балансировка классов
     - C=1.0 - параметр регуляризации

5. **Оценка качества моделей:**
   - Разделение данных на обучающую (80%) и тестовую (20%) выборки
   - Оценка метрик: accuracy, precision, recall, F1-score
   - Выбор модели Logistic Regression как оптимальной (точность ~98.82%)

6. **Сохранение моделей:**
   - Сохранение обученной модели в формате .pkl
   - Сохранение векторизатора в формате .pkl
   - Размещение файлов в директории modelN/

### 3.2 Разработка веб-приложения и REST API

Веб-приложение разработано на Flask и предоставляет как пользовательский интерфейс, так и REST API для интеграции с другими системами.

**Основные маршруты Flask приложения:**
- GET / - главная страница веб-приложения
- GET /webapp - страница для Telegram Web App
- POST /api/predict/text - анализ текста на фишинг
- POST /api/predict/image - анализ изображения (с OCR)
- POST /api/predict/eml - анализ .eml файла
- GET /api/health - проверка работоспособности API

**Реализация обработки различных типов данных:**
- **Текст:** извлечение признаков (URL, email, телефон), очистка, векторизация, предсказание
- **Изображения:** извлечение текста с помощью Tesseract OCR, затем обработка как текста
- **.eml файлы:** парсинг email-файла, извлечение тела письма и заголовков, обработка как текста

**Особенности реализации:**
- Использование Flask-CORS для поддержки CORS при работе с Chrome расширением
- Обработка ошибок с детальными сообщениями для пользователя
- Проверка доступности OCR перед использованием
- Очистка временных файлов после обработки
- Ограничение размера загружаемых файлов (16 MB)

### 3.3 Разработка Telegram бота и интеграции с Web App

Telegram бот разработан с использованием библиотеки python-telegram-bot и интегрирован с Telegram Web App API.

**Основные функции бота:**
- Команда /start - отправка приветственного сообщения с кнопкой для открытия Web App
- Команда /help - отправка справки по использованию бота
- Inline кнопка "Открыть проверку фишинга" - открывает Telegram Web App

**Интеграция с Telegram Web App:**
- Настройка WEBAPP_URL на HTTPS URL развернутого веб-приложения
- Использование InlineKeyboardButton с параметром web_app для создания кнопки Web App
- Telegram Web App работает полностью внутри Telegram, не требуя перехода на внешний сайт

**Требования для работы Web App:**
- Обязательное использование HTTPS (Telegram требует только HTTPS ссылки)
- Развертывание на облачной платформе (Railway.app, Render.com и т.д.)
- Настройка правильного URL в коде бота

### 3.4 Разработка Chrome расширения

Chrome расширение разработано по стандарту Manifest V3 и предоставляет возможность проверки веб-страниц на фишинг прямо из браузера.

**Структура расширения:**
- manifest.json - конфигурация расширения с разрешениями и метаданными
- popup.html - HTML интерфейс popup-окна
- popup.js - JavaScript логика для извлечения текста и отправки запросов к API
- content.js - content script для извлечения текста со страницы
- background.js - background service worker (при необходимости)

**Функциональность:**
- При нажатии на иконку расширения извлекается текст со страницы
- Текст автоматически очищается от HTML-тегов и скриптов
- Извлекаются URL, email и телефоны из текста
- Отправляется запрос к API (POST /api/predict/text)
- Результаты отображаются в popup-окне с визуальной индикацией уровня риска

**Особенности реализации:**
- Использование chrome.scripting.executeScript для извлечения текста
- Обработка ошибок при недоступности API
- Настройка API URL через chrome.storage.local
- Поддержка как локального API (localhost), так и развернутого на сервере

### 3.5 Интеграция компонентов системы

Все компоненты системы интегрированы через единый REST API, что обеспечивает консистентность результатов и упрощает поддержку.

**Схема взаимодействия компонентов:**
- Веб-приложение использует модуль машинного обучения напрямую через импорт класса PhishingModel
- Telegram Web App отправляет запросы к веб-приложению через HTTPS
- Chrome расширение отправляет запросы к API веб-приложения
- Все компоненты используют одинаковые endpoints API для получения результатов

**Обработка ошибок:**
- Проверка доступности модели при запуске приложения
- Graceful fallback при отсутствии Tesseract OCR
- Детальные сообщения об ошибках для пользователя
- Логирование ошибок для отладки

**Оптимизация производительности:**
- Ленивая загрузка модели (при первом запросе)
- Кэширование векторизатора
- Использование Gunicorn для продакшена
- Stateless API (возможность горизонтального масштабирования)
